# cf_ai_miku_teto_battle

A Friday Night Funkin-style rhythm battle game featuring Hatsune Miku vs Kasane Teto, built entirely on Cloudflare's developer platform.

**Live:** https://cf-ai-miku-teto-battle.tony-e-bolivar.workers.dev

## How to Play

1. Pick your character (Miku or Teto)
2. Choose Bot mode or PVP multiplayer
3. Select a song — your opponent delivers AI-generated trash talk in Japanese before the match
4. Hit arrow keys (or WASD) in time with the notes scrolling down your highway
5. Outscore your opponent to win!

## AI Components

### LLM — Workers AI (Llama 3.3 70B)

Before each match, the opponent character delivers trash talk generated by **Llama 3.3 70B** running on Workers AI. The response streams token-by-token into a speech bubble. The opponent stays in character as Miku or Teto and speaks in Japanese.

- **Model:** `@cf/meta/llama-3.3-70b-instruct-fp8-fast`
- **Endpoint:** `POST /api/trash-talk`
- **Streaming:** SSE response parsed client-side for real-time typing effect

### Workflow / Coordination — Durable Objects

Real-time PVP multiplayer is powered by a **Durable Object** (`LobbyDO`) that coordinates the full match lifecycle via WebSocket hibernation API:

- Lobby creation with shareable 6-character codes
- Player slot assignment, ready-up, and countdown sync
- Server-authoritative health/score calculation during gameplay
- Real-time opponent state broadcast (score, combo, animations)
- PVP chat relay before and after matches

### User Input — Chat & Gameplay

- **PVP Chat:** Real-time text chat between players via the Durable Object WebSocket
- **Gameplay:** 4-lane keyboard input judged against timing windows, with hit/miss events sent to server in PVP

### Memory / State

- **Durable Object Storage:** Match state persisted across WebSocket hibernation cycles
- **KV Namespace:** Leaderboard scores stored persistently
- **Client State:** Health, score, combo tracked with event-driven UI updates

## Stack

| Layer | Technology |
|---|---|
| Frontend | React + TypeScript + Three.js |
| Game Engine | Canvas 2D (note highways) + Three.js (3D characters, stages) |
| 3D Characters | @pixiv/three-vrm for VRM models, three-stdlib MMDLoader for PMX models with VMD dance animation |
| Lobby State | Durable Objects (WebSocket Hibernation API) |
| Matchmaking API | Cloudflare Workers |
| Trash Talk | Workers AI (Llama 3.3 70B, streaming) |
| Leaderboard | Cloudflare KV |

## Songs

- **Po Pi Po** (138 BPM) - Hatsune Miku, with VRM character models and GLB beach stage
- **Mesmerizer** (143 BPM) - Hatsune Miku & Kasane Teto SV, with PMX models dancing via VMD motion data and video background

## Game Mechanics

- 4 lanes: Left (purple), Down (cyan), Up (green), Right (red)
- Hit windows: SICK +-45ms, GOOD +-90ms, BAD +-135ms, SHIT +-166ms
- Zero-sum health bar (0.0-2.0), KO at 0.0
- Hold notes, combo tracking, rating popups
- Bot opponent with 3 difficulty levels
- Japanese lyrics with English translation synced to SRT timing
- Web Audio API for precise timing + input buffering
- Per-song custom assets: VRM/PMX models, VMD dance animations, GLB stages, video backgrounds

## Local Development

```bash
npm install
npm run dev
```

Open http://localhost:5173

### Dev Tools (query params)

- `?startAt=30000` - skip to 30s into the song
- `?slowmo=0.5` - half-speed playback
- `?hitwindow=1` - visualize hit windows

## Deployment

```bash
npx wrangler login
npm run build
npm run deploy
```

Workers AI trash talk and KV leaderboard require Cloudflare bindings (already configured in `wrangler.jsonc`). Locally, trash talk falls back to hardcoded Japanese lines.
